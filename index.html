The AI Wrapper Dilemma

Every AI company today faces the same challenge: start fast with APIs, or spend years building proprietary models before launch?

The Wrapper Trap

  • High API costs: ¬£0.02-0.05 per analysis adds up quickly at scale
  • No competitive moat: Anyone can call the same APIs
  • Dependency risk: Pricing changes, rate limits, service outages
  • No IP ownership: Core technology belongs to OpenAI, Anthropic, etc.

The Build Trap

  • Slow to market: 12-18 months to collect data and train models
  • High upfront cost: ML infrastructure, data scientists, compute
  • Uncertain ROI: No guarantee the model will work before launch
  • Chicken-and-egg: Need users to generate data, need data to build product

There's a Third Way

Start as a wrapper. Build a moat through data. Transition to proprietary models.

Automating compliance while capturing decision-making intelligence

How It Works

  1. Fetch: Agent retrieves application data from database
  2. Query: Uses RAG to search lender policy documents for relevant rules
  3. Analyse: GPT-4 evaluates application against compliance requirements
  4. Report: Generates structured audit with pass/fail recommendations
  5. Review: Human broker validates or corrects AI decisions

Tech: LangChain, GPT-4, FAISS, PyPDF

Pillar 2: Visual Forensic Analyser

Pre-application fraud detection while building fraud pattern recognition

1. D2D Extraction

GPT-4o Vision extracts structured data from document images. Creates training data by capturing all visible fields and values as the model "sees" them.

2. Math Validation

Verifies calculation accuracy (gross pay = basic + overtime, net pay after deductions). Flags arithmetic inconsistencies that indicate manipulation.

3. Spatial Fingerprinting

SSIM compares layout against gold standard template. Detects layout deviations and format changes.

4. Error Level Analysis

Detects compression artifacts indicating digital manipulation. Identifies regions with different compression levels revealing altered areas.

5. Risk Scoring

LLM synthesizes all findings into comprehensive fraud probability with clear pass/fail recommendations and detailed reasoning.

Human Review

Experts validate or correct AI findings. Changes analysed by GPT-4o to understand model failures, creating labelled training data.

Training Data Captured

  • Extracted field values + OCR confidence scores
  • SSIM similarity metrics + layout deviation heat maps
  • ELA compression analysis + manipulation region coordinates
  • Human forensic expert corrections + fraud type labels

Exports: raw_analysis_*.json (automatic) + forensic_analysis_*.json (human-reviewed)

The Training Data Challenge

To deploy pre-application fraud detection, we need labelled fraud examples. Solution: manual fraud sample creation by security experts.

  • Create 100-200 realistic forgeries (altered values, fake logos, font mismatches)
  • Label each with specific manipulation types
  • Combine with legitimate documents for balanced training

Future: Specialised Fraud Detection Model

Fine-tune vision model (e.g., Phi-3 Vision) on UK payslip/bank statement fraud patterns. Outperforms GPT-4o Vision for this specific domain.

Cost per analysis: ~¬£0.001-0.003

Speed: 2-3x faster

Higher accuracy on UK-specific fraud patterns

Tech: GPT-4o Vision, OpenCV, scikit-image, NumPy
